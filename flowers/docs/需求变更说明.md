# 花店智能管理系统需求变更说明

发布日期：2026-03-10  
适用范围：商家端（Web）、管理员端（Web）、用户端（微信小程序）、后端接口（Spring Boot）

## 一、变更背景

为提升库存管理可视化、评论管理合规性以及用户下单前门店选择体验，现对系统功能做如下调整。

## 二、变更内容

### 1）库存预警：预警商品信息可视化

- 商家端「库存预警」页面的库存列表新增字段：
  - 最近进货时间（最近一次入库时间）
  - 最早过期时间（最先到期批次的到期时间，并展示剩余天数）
- 导出预警列表（CSV）同步新增上述字段，便于线下留存与核对。

**验收要点**
- 列表可直观看到预警商品的进货/过期时间信息；
- 展开批次明细时，“最早过期时间”与 FEFO 批次排序逻辑一致。

### 2）评论权限：商家不可删除评论，管理员可删除

- 商家端「评论管理」移除“删除评论”操作，仅保留“回复/修改回复”。
- 后端删除评论接口增加权限限制：仅允许管理员（SUPER_ADMIN）执行删除。

**验收要点**
- 商家账号无法删除评论（前端无入口，后端接口也会拒绝）；
- 管理员账号可以删除评论。

### 3）用户端：门店地址展示与花店选择

- 后端新增公开接口：获取可用门店列表（含门店地址/电话等信息）。
- 用户端小程序新增「选择花店」页面：
  - 展示门店名称、门店地址、联系电话
  - 支持选择门店后筛选展示该门店商品
- 首页/分类/搜索页增加门店筛选入口；商品详情页展示门店地址。

**验收要点**
- 用户可在小程序中查看门店地址，并完成门店切换；
- 门店切换后商品列表随之变化；详情页可展示门店地址信息。

## 三、影响与说明

- 涉及模块：
  - 商家端：库存预警页、评论管理页
  - 用户端：门店选择页、首页/分类/搜索、商品详情
  - 后端：门店公开查询接口、评论删除权限校验
- 如门店数据为空：用户端将显示空状态提示，需先在后台维护门店信息。
