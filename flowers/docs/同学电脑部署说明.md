# 同学电脑部署说明

## 1. 运行环境

推荐最小环境：
- Windows 10/11
- Java 17
- MySQL 8.x

如果需要在同学电脑重新构建前端，再额外安装：
- Node.js 18+

## 2. 项目结构要求

部署时请保持以下目录结构不变：
- `flowers/backend`
- `flowers/flower-admin-web`
- `flowers/flower-merchant-web`
- `flowers/sql`
- `flowers/scripts`

后端会直接读取：
- `../flower-admin-web/dist`
- `../flower-merchant-web/dist`

所以不要改目录名。

## 3. 数据库初始化

推荐直接执行一份初始化脚本：
1. 执行 `sql/init.sql`

这份文件来自当前可运行数据库的完整导出，已经包含：
- 表结构
- 管理员/商家认证数据
- 商品、用户、商家、订单、评价等演示数据

## 4. 默认数据库配置

后端默认连接：
- 库名：`flower_shop`
- 地址：`localhost:3306`
- 用户：`root`
- 密码：`root`

配置文件位置：
- `backend/src/main/resources/application.yml`

也可以通过环境变量覆盖：
- `DB_URL`
- `DB_USERNAME`
- `DB_PASSWORD`
- `ADMIN_TOKEN`
- `ADMIN_ACCOUNT`
- `ADMIN_PASSWORD`
- `MERCHANT_ACCOUNT`
- `MERCHANT_PASSWORD`

## 5. 前端处理方式

### 如果直接拷贝当前仓库
当前仓库已经包含构建后的 `dist`，可以直接启动后端。

### 如果要在同学电脑重新构建
先分别执行：

管理员端：
```powershell
cd flower-admin-web
npm install
npm run build
```

商家端：
```powershell
cd flower-merchant-web
npm install
npm run build
```

## 6. 启动后端

```powershell
cd backend
mvn -q -DskipTests package
java -jar target/flower-shop-backend-1.0.0.jar
```

默认启动地址：
- `http://127.0.0.1:18080`

## 7. 登录地址

管理端：
- `http://127.0.0.1:18080/admin/login`
- `http://127.0.0.1:18080/admin/register`
- `http://127.0.0.1:18080/admin/change-password`

商家端：
- `http://127.0.0.1:18080/merchant/login`
- `http://127.0.0.1:18080/merchant/register`
- `http://127.0.0.1:18080/merchant/change-password`

默认账号：
- 管理员：`admin / admin123456`
- 商家：`merchant / merchant123456`

## 8. 冒烟验证

后端启动后，建议直接运行：

```powershell
powershell -ExecutionPolicy Bypass -File scripts/db-smoke-check.ps1 -BaseUrl http://127.0.0.1:18080/api/v1
```

这个脚本会验证：
- 用户写入数据库
- 商品读取成功
- 购物车写入成功
- 订单写入成功
- 支付后状态更新成功
- 用户信息更新成功

## 9. 我这边已验证通过的内容

已经确认：
- 前端页面能打开
- 认证注册、登录、修改密码都能写入数据库
- 商品、用户、商家、订单接口都能正常读取
- 新注册账号在后端重启后仍然可以登录，说明数据已持久化到 MySQL

## 10. 如果同学电脑启动失败

优先检查：
1. MySQL 是否启动
2. `flower_shop` 数据库是否已导入
3. 18080 端口是否被占用
4. `flower-admin-web/dist` 和 `flower-merchant-web/dist` 是否存在
5. Java 版本是否为 17
