# 系统架构说明（可用于开题/中期/答辩）

## 1. 架构总览

系统采用四层架构：

1. 展示层：微信小程序（用户端） + Vue 3 管理后台（商家端/管理员端）
2. 业务层：Spring Boot REST API
3. 数据层：MySQL 8.x
4. 分析层：Python 预测与推荐服务

```text
用户小程序        商家后台(Vue 3)        管理员后台
    \                |                  /
             ------ REST API ------
                 Spring Boot
     (用户/商品/订单/库存/评价/报表)
                 |               \
               MySQL         Python分析服务
                 |               |
            业务明细表       预测与推荐结果表
```

## 2. 业务闭环

系统围绕订单主线构建完整业务闭环：

用户下单 -> 库存锁定 -> 支付成功 -> 正式扣减库存 -> 履约完成 -> 售后评价 -> 数据分析 -> 推荐与补货建议。

其中库存一致性流程如下：

- 下单：锁定库存（`LOCKED`）
- 支付：锁定转扣减（`CONFIRMED`）
- 取消：释放锁定库存（`RELEASED`）
- 退款：回补已扣减库存（`ROLLED_BACK`）

## 3. 角色与权限边界

- 用户端：商品浏览、下单支付、订单跟踪、售后评价
- 商家端：商品维护、BOM 管理、订单处理、库存预警、销售统计
- 管理员端：商家管理、用户管理、角色权限、日志审计、参数配置

## 4. 关键数据模型

- 商品模型：`product`
- 花材模型：`flower_material`
- 组合关系：`product_bom`
- 批次库存：`inventory_batch`
- 订单主表：`customer_order`
- 订单明细：`order_item`
- 锁库存流水：`stock_lock`
- 预测结果：`forecast_result`
- 补货建议：`replenishment_suggestion`
- 推荐结果：`recommendation_result`

## 5. 智能模块落地方式

- 预测：优先 Prophet，数据量不足时自动退化为移动平均
- 补货：`建议补货量 = max(0, 预测需求 + 安全库存 - 现有库存 - 在途量)`
- 推荐：用户偏好 + 近期热销 + 库存可用性的混合打分
